The programming language that powers StudyMate AI

Key Language Features Used

  • Null Safety — Sound null safety, ?, !, late
  • Records — (x, y) multi-return without classes
  • Pattern Matching — switch + destructuring
  • Sealed Classes — Exhaustive type hierarchies
  • Extension Methods — Add methods to any type

  • Mixins — Reusable behavior composition
  • Collections — Spread, null-aware, collection-if

Why Dart?

  • AOT → Native machine code (fast startup)
  • JIT → Hot reload (fast development)
  • Single language → UI + logic + backend
  • Strong typing → Fewer runtime bugs
  • Rich stdlib → dart:io, dart:convert, dart:math
  • Isolates → True parallelism with shared nothing
  • Tree-shaking → Minimal APK size
  • Google-backed → Long-term stability

🔮 Riverpod 2.6.1

State management — compile-safe, testable, scalable

Provider Types Used

  • Provider — Pure computed values (e.g. levelFromXp)
  • StateProvider — Simple mutable state (e.g. currentTab)
  • StateNotifierProvider — Complex state with logic (e.g. AuthNotifier)
  • FutureProvider — Async data (e.g. quiz list)
  • StreamProvider — Real-time data (e.g. study timer)
  • Family — Parameterized providers (e.g. quizBySubject)
  • AutoDispose — Auto-cleanup when not watched

Why Riverpod over Provider?

  • Compile-time safe — No runtime ProviderNotFoundException
  • No BuildContext needed — Access state anywhere
  • Auto-dispose — Prevents memory leaks
  • Testable — Override any provider in tests
  • Family — Create providers with parameters
  • Ref.watch/listen/read — Granular rebuild control
// Example: Gamification provider final xpProvider = StateNotifierProvider<XpNotifier, int>((ref) { return XpNotifier(ref.read(dbProvider)); }); final levelProvider = Provider<int>((ref) { final xp = ref.watch(xpProvider); return (sqrt(xp / 100)).floor(); // N² × 100 formula });

🗄️ SQLite (sqflite 2.4.3)

Embedded SQL database — zero configuration, full SQL support

Database Configuration

  • WAL Mode — Write-Ahead Logging for concurrent reads/writes
  • FTS5 — Full-text search on notes and content
  • Foreign Keys — Referential integrity enforced
  • Auto-vacuum — Automatic space reclamation
  • Custom Migrations — Versioned schema updates
  • 6 Repositories — User, Education, Study, Content, Gamification, Social
DatabaseHelper class manages all connections

Dual-Write Strategy

  • SharedPreferences — Immediate synchronous reads/writes (JSON-serialized)
  • SQLite — Asynchronous background writes for durability
  • AppDatabase — Unified API that abstracts both storage layers
  • Migration Plan — All future data goes SQLite-only
Offline-first — no internet required for any core feature

🎮 Gamification Engine

Built from scratch — 13 XP events, quadratic levels, 23 achievements

Level Formula

XP Required = N² × 100

  • Level 1: 0 XP  |  Level 10: 10,000 XP
  • Level 5: 2,500 XP  |  Level 20: 40,000 XP
  • Level 50: 250,000 XP  |  Cap: Level 100
13 XP event types trigger on user actions

Achievement System

  • Auto-checked — On every XP event, all 23 achievements re-evaluated
  • Progress tracking — Each achievement has target number
  • Unlock notification — Popup when achievement is earned
  • Categories: Study, Quiz, Social, Streak, XP, Milestone, Special
// Gamification core logic int levelForXp(int xp) => (sqrt(xp / 100)).floor(); int xpForLevel(int level) => level * level * 100; double xpProgress(int xp) { final lvl = levelForXp(xp); final cur = xp - xpForLevel(lvl); final next = xpForLevel(lvl + 1) - xpForLevel(lvl); return cur / next; // 0.0 → 1.0 progress bar }

🃏 FSRS Algorithm

Free Spaced Repetition Scheduler — next-gen flashcard review

4 Rating Levels

  • Again — Total recall failure, reset interval
  • Hard — Recalled with difficulty, shorter interval
  • Good — Correct recall, normal progression
  • Easy — Effortless recall, longest interval jump

Core Metrics Tracked

  • Stability (S) — How long the card will retain memory (days)
  • Difficulty (D) — How inherently hard the card is (1-10)
  • Retrievability (R) — Probability of recall today (0-100%)
  • Next Review — Calculated based on S × desired R threshold
// FSRS state for each flashcard class FlashcardFsrs { final double stability; // S — interval in days final double difficulty; // D — 1 (easy) to 10 (hard) final double retrievability; // R — 0.0 to 1.0 final DateTime nextReview; FlashcardFsrs advance(int rating) { // rating: 1=again, 2=hard, 3=good, 4=easy // Update stability and difficulty based on rating // Calculate new nextReview = now + new stability } }
